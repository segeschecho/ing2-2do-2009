\chapter{Impediment List}

En este capitulo se mostrará la lista $Impediment$ $List$ con los problemas que pueden surgir en el proyecto y que afecte a su productividad y calidad. Estos estarán agrupados por los $User$ $Story$ que se realizarán en la primera iteración.

\begin{enumerate}
\item US03: Trabajar en colaboración con otras regiones (modelos)

\begin{itemize}
\item Algun problema realizando el protocolo de comunicación.
\item Posibilidad de realizar mas cambios que lo planeado en la arquitectura para posibilitar la comunicación y el intercambio de informacion.
\item Posibles problemas en la implementación por uso de lenguajes o tecnicas desconocidas.
\item Aumento en el costo del testeo de los nuevos componentes y detección de posibles errores en la arquitectura.
\end{itemize}

\item US08: Suscrpción de modelos a Trs

\begin{itemize}
\item Problemas definiendo las interfaces y el metodo de suscripción de los modelos.
\item Posibilidad de realizar mas cambios que lo planeado en la arquitectura.
\item Posibles problemas implementando la suscripción de estaciones centrales a trs.
\item Localización de errores en la arquitectura a la hora de realizar los testeos.
\item Problemas para definir correctamente la partición de las reglas.
\item Problemas realizando la unificación de las particiones.
\item Posibles problemas para solucionar el problema de la colaboración distribuida entro los modelos.
\item Complicaciones realizando el diagrama de objetos relacionado con la ejecución de los modelos matematicos.
\end{itemize}

\item US10: Configuración sencilla de colaboración entre modelos

\begin{itemize}
\item Problemas para resolver la configuración que permita la colaboración entre modelos.
\item Localización de errores en la arquitectura a la hora de realizar los testeos.
\end{itemize}


\item US11: Varios algoritmos de evaluación de reglas

\begin{itemize}
\item Posibles complicaciones realizando el diagrama de objetos para los algoritmos de evaluación de reglas.
\end{itemize}

\end{enumerate}
